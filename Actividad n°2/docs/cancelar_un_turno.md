# Escenario de caso de uso 4

# Cancelar un turno

## Descripción General
El usuario puede cancelar un turno cuando necesite

## Flujo de eventos
* El usuario ingresa al sistema con sus datos.
* El sistema verifica la información.
* El usuario ingresa al apartado de Turnos - Cancelar turno.
* El sistema pide el código del turno.
* El usuario ingresa el código de seguimiento.
* El sistema verifica el código y da acceso al apartado de cancelar turno.
* El usuario da clic en el apartado de Cancelar turno
* El sistema imprime un mensaje de "Seguro que quieres cancelar el turno" y da dos opciones "Si" o "No"
* La selección de usuario en el apartado de "Si"
* El sistema muestra un mensaje de "Turno cancelado"
* El sistema elimina el turno.

 # Precondiciones:
 El usuario debe tener un turno activo.

# Postcondiciones:
El turno se elimina del sistema.
